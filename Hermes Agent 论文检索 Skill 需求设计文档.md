# Hermes Agent 论文检索 Skill 需求设计文档

## 1 文档概述

### 1\.1 编写目的

本文档为 Hermes Agent 专属**论文检索 Skill** 的标准化需求设计文档，明确该智能体技能的业务背景、能力边界、触发规则、核心功能、输入输出规范、运行约束及验收标准。为后续 Skill 提示词编写、工具对接、逻辑开发、测试上线提供唯一依据，解决新手无规范、边界模糊、功能混乱的开发问题，保障技能可复用、可迭代、可工程化落地。

### 1\.2 适用范围

本文档适用于 Hermes Agent 论文检索技能的设计、开发、调试、测试、迭代全流程，面向 Agent 开发人员、测试人员及技能运维人员。

### 1\.3 核心定义

- **Hermes Agent**：轻量化可扩展 AI 智能体，支持模块化 Skill 挂载，通过独立 Skill 封装单一业务能力

- **Skill（技能）**：Agent 可独立调用的模块化能力单元，单职责、可复用、可自动触发，区别于单次临时 Prompt

- **论文检索 Skill**：专门承接学术论文查询、筛选、整理、汇总的专属 Agent 能力模块

## 2 业务需求与背景

### 2\.1 业务背景

学术研究、课题调研、文献综述撰写过程中，用户需要高频检索中英文学术论文，包含主题查新、关键词检索、文献筛选、核心信息提取、参考文献整理等需求。原生 Agent 对话模式无标准化检索流程，存在检索杂乱、信息不全、筛选无规则、输出格式混乱、无法精准匹配学术场景等问题。因此需要独立开发**论文检索专属 Skill**，让 Hermes Agent 可自动识别论文检索需求、标准化执行检索流程、规整输出学术文献结果。

### 2\.2 目标用户

学生、科研人员、课题研究者、需要撰写论文/综述的学术用户

### 2\.3 核心业务目标

- 让 Hermes Agent 可精准识别用户各类论文检索指令，自动触发检索技能

- 标准化完成论文检索、筛选、去重、信息提取、格式整理全流程

- 输出符合学术规范的论文结果，支持后续文献综述、参考文献复用

- 实现能力模块化，不干扰 Agent 其他对话、办公、问答技能，满足单职责设计原则

## 3 技能边界与适用范围（核心）

明确 Skill 可用场景、禁用场景，避免 Agent 误触发、错调用，是 Agent Skill 需求设计的核心要点。

### 3\.1 适用场景（触发条件）

当用户提问包含以下任意场景，Agent **必须触发本论文检索 Skill**，不再执行普通对话逻辑：

- 用户明确要求检索、查找、搜索相关学术论文、期刊文献、学位论文、会议论文

- 用户提供研究主题、关键词、研究方向，需要匹配相关参考文献

- 用户需要查新、查找近年度最新研究论文、高被引论文、核心期刊论文

- 用户需要筛选特定条件论文（如：近3年、SCI/核心期刊、高引用、英文文献）

- 用户需要整理检索到的论文核心摘要、研究亮点、创新点、参考文献格式

### 3\.2 不适用场景（禁用边界）

以下场景 **禁止触发本 Skill**，Agent 走普通对话或其他技能逻辑：

- 非学术论文检索（普通新闻、博客、科普文章、网文资料检索）

- 论文写作润色、查重、降重、改写、翻译全文等文本编辑需求

- 专利、书籍、报告、课件等非期刊/会议/学位论文检索

- 无明确检索意图的学术闲聊、概念咨询

## 4 功能需求（核心功能清单）

本 Skill 遵循**单职责原则**，聚焦论文检索全流程，包含 5 大核心功能，所有功能均可被 Agent 自动调度执行。

### 4\.1 需求解析功能

自动解析用户检索指令，提取核心检索参数，无需用户重复补充信息。

- 自动提取：检索主题、核心关键词、研究领域、语种（中文/英文）

- 自动识别：时间范围（近1年/3年/5年/不限）、文献类型（期刊/会议/学位论文）

- 自动识别：筛选条件（高被引、核心期刊、SCI、EI、最新研究）

- 缺省补全：用户未指定条件时，默认检索近3年中英文核心相关论文

### 4\.2 智能检索功能

基于解析后的参数，执行标准化学术论文检索，支持多维度精准匹配。

- 支持主题模糊检索、关键词精准检索、领域定向检索

- 支持中英文文献混合检索、单语种定向检索

- 自动过滤低质量、重复、非学术无关文献

### 4\.3 文献筛选与排序功能

对检索结果进行智能筛选、优先级排序，输出高价值文献。

- 优先级：高被引论文 \> 核心/SCI/EI 期刊论文 \> 顶会论文 \> 普通期刊论文

- 支持按发布时间倒序、引用量降序排序

- 自动剔除重复文献、资讯类非学术文献

### 4\.4 论文信息提取与整理功能

对筛选后的论文，标准化提取核心信息，规整输出内容。单篇论文固定提取字段：

- 论文标题、作者、发表期刊/会议、发表年份

- 论文摘要、核心研究内容、创新点

- 研究价值与应用场景、核心结论

- 标准参考文献引用格式（GB/T 7714）

### 4\.5 结果汇总与答疑功能

- 自动汇总检索结果，统计文献数量、文献分布（年份、期刊类型）

- 支持基于检索结果解答用户延伸问题（如该领域研究现状、研究热点、空白方向）

## 5 输入输出需求规范

### 5\.1 输入规范

输入来源：用户自然语言提问（无固定格式）

输入内容示例：

- 检索近三年大模型在教育领域的核心论文

- 查找人工智能图像分割的高被引英文论文

- 推荐几篇物联网安全的最新学位论文

### 5\.2 输出规范（固定格式，强制标准化）

输出结构分为三部分，统一学术风格，条理清晰，可直接用于文献综述撰写。

1. **检索概况**：检索主题、筛选条件、检索时间、有效文献总数

2. **核心论文列表**：按优先级排序，单篇包含完整提取字段

3. **检索总结**：领域研究现状、核心研究方向、研究热点与空白

#### 5.3.2 PDF 版式规范

- **页面设置**：A4纸张，上下左右页边距 2.5cm
- **字体规范**：
  - 标题：思源黑体 / Microsoft YaHei UI，Bold 14-18pt
  - 正文：思源宋体 / SimSun，Regular 10-12pt
  - 英文内容：Times New Roman / Calibri，Regular 10-12pt
  - 代码/DOI：Consolas / Courier New，Regular 9pt
- **行距**：1.5 倍行距，段前段后各 6pt
- **页眉页脚**：
  - 页眉：左侧"{研究领域}论文检索报告"，右侧"第X页/共Y页"
  - 页脚：中间"Generated by Hermes Agent | {检索日期}"
- **配色方案**：
  - 主色调：学术蓝 #2C5F8D（用于标题、分割线）
  - 辅助色：数据灰 #7F8C8D（用于统计图表）
  - 强调色：亮点橙 #E67E22（用于推荐理由标签）

#### 5.3.3 PDF 生成技术要求

- **PDF 生成引擎**：推荐使用 ReportLab (Python) 或 WeasyPrint（支持 CSS 布局）
- **图表渲染**：使用 Matplotlib 生成趋势图，使用 NetworkX 生成知识图谱
- **中文支持**：确保字体文件完整嵌入（思源黑体/宋体），避免乱码
- **元数据嵌入**：PDF Properties 中包含 Title、Author、Keywords、CreationDate
- **可访问性**：设置 PDF 文档结构标签，支持屏幕阅读器
- **文件大小优化**：图片压缩至 150 DPI，总文件大小控制在 5MB 以内

### 5\.4 输入参数规范表

为强化论文检索 Skill 的参数规范，确保用户意图被正确解析，制定以下标准化参数表：

#### 5.4.1 核心检索参数表

| 参数类别 | 参数名称 | 数据类型 | 必需性 | 默认值 | 可选值/示例 | 参数说明 | 解析规则 |
|---------|---------|---------|-------|-------|-----------|---------|---------|
| **检索主题** | `research_topic` | string | 必需 | 无 | "大模型在教育领域应用"、"物联网安全" | 用户研究的核心主题 | 从用户输入第一句中提取核心名词短语 |
| **关键词** | `keywords` | array[string] | 可选 | 从主题自动提取 | ["LLM", "education", "pedagogy"] | 精准检索关键词列表 | 支持中英文混合，最多10个 |
| **时间范围** | `time_range` | object | 可选 | 近3年 | 见下表详细定义 | 论文发表时间筛选 | 识别"近X年"、"XXXX-XXXX"、"最新"等表述 |
| **文献类型** | `paper_type` | array[string] | 可选 | ["journal", "conference"] | 见下表详细定义 | 目标文献类型组合 | 识别"期刊"、"会议"、"学位"等关键词 |
| **语种偏好** | `language` | string | 可选 | " bilingual" | "zh", "en", "bilingual" | 目标论文语言 | 识别"中文"、"英文"、"中英"等 |
| **筛选条件** | `filters` | object | 可选 | 见默认值 | 见下表详细定义 | 高级筛选条件集合 | 识别"高被引"、"SCI"、"核心"等质量要求 |
| **排序方式** | `sort_by` | string | 可选 | "relevance" | "citation_count", "publish_date", "relevance" | 结果排序依据 | 识别"按引用"、"按时间"、"最相关"等 |
| **输出数量** | `max_results` | integer | 可选 | 8 | 5-20 | 返回论文数量上限 | 识别"X篇"、"几个"等数量表述，最大不超过50 |

| 参数值 | 含义 | 解析关键词示例 | 日期范围计算 | 适用场景 |

| `"1y"` | 近1年 | "近一年"、"去年"、"recent" | 当前日期-365天 | 快速查新、最新趋势 |

#### 5.4.3 文献类型参数子表

| 参数值 | 含义 | 识别关键词 | 数据源优先级 | 说明 |
|-------|------|-----------|------------|------|
| `"journal"` | 期刊论文 | "期刊"、"journal"、"article" | 1. SCI/EI 2. 核心期刊 3. 普通期刊 | 同行评审，质量较高 |
| `"conference"` | 会议论文 | "会议"、"conference"、"proceedings" | 1. CCF-A类 2. CCF-B类 3. 顶会 4. 普通会议 | 前沿技术，更新快 |
| `"thesis"` | 学位论文 | "学位论文"、"thesis"、"dissertation" | 1. 博士论文 2. 优秀硕士论文 | 研究系统，篇幅长 |
| `"preprint"` | 预印本 | "预印本"、"preprint"、"arXiv" | arXiv、bioRxiv 等预印本平台 | 最新成果，未经同行评审 |

2. 时间范围：识别"近X年"等表述，默认"3y"
3. 文献类型：识别"期刊/会议/学位"，默认["journal", "conference"]
4. 筛选条件：识别"高被引/SCI/核心"，默认{min_citation_count: 10, require_core_journal: true}

- 用户说"SCI/EI" → 对应布尔标记设为 true
- 用户说"最新" → sort_by: "publish_date", time_range: "1y"

```

## 6 数据源层次与优先级定义

为确保论文检索结果的质量和可获取性，建立数据源分层体系，定义优先级规则和身份认证流程。

### 6\.1 数据源分层体系

#### 6.1.1 数据源层次金字塔

```
        ┌─────────────────────────────────────────┐
        │      Tier 1: 顶级权威数据源 (最高优先)      │
        │   国际顶级期刊出版商 + 国际权威学术数据库    │
        ├─────────────────────────────────────────┤
        │      Tier 2: 核心中文数据源 (次高优先)      │
        │    国内核心期刊数据库 + 中文学术平台       │
        ├─────────────────────────────────────────┤
        │      Tier 3: 补充数据源 (一般优先)        │

### 6\.6 数据源优先级调度规则

**智能检索策略**（由 Agent 自动执行）：

```python
# 伪代码：数据源优先级调度逻辑
def retrieve_papers(user_params):
    results = []
    exhausted_tiers = []
    
    # 尝试 Tier 1 数据源
    if user_has_institution_access(user_params):
        tier1_sources = ["WoS", "Scopus", "IEEE", "Nature", "ScienceDirect"]
        for source in tier1_sources:
            try:
                results.extend(query_source(source, user_params))
                if len(results) >= user_params.max_results:
                    return rank_and_deduplicate(results)
            except AuthenticationError:
                exhausted_tiers.append((source, "需要机构认证"))
            except QueryError:
                continue
    
    # 尝试 Tier 2 数据源（中文优先）
    if user_params.language in ["zh", "bilingual"]:
        tier2_sources = ["CNKI", "万方", "维普"]
        for source in tier2_sources:
            try:
                results.extend(query_source(source, user_params))
                if len(results) >= user_params.max_results:
                    return rank_and_deduplicate(results)
            except AuthenticationError:
                exhausted_tiers.append((source, "需要机构认证"))
    
    # 尝试 Tier 3 数据源（开放获取）
    tier3_sources = ["arXiv", "PMC", "DOAJ"]
    for source in tier3_sources:
        results.extend(query_source(source, user_params))
    
    # 尝试 Tier 4 数据源（开放网络）
    if len(results) < user_params.max_results:
        tier4_sources = ["Google Scholar", "Semantic Scholar"]
        for source in tier4_sources:
            results.extend(query_source(source, user_params))
    
    return rank_and_deduplicate(results)
```

**降级策略**：
- 如果 Tier 1 需要认证且用户无法提供 → 自动降级至 Tier 2
- 如果 Tier 2 需要认证且用户无法提供 → 自动降级至 Tier 3
- Tier 3（开放获取）和 Tier 4（开放网络）**无需认证**，作为保底选项

### 6\.7 身份认证安全提示

**数据隐私承诺**：
- 用户提供的机构信息和凭据**仅用于单次检索**
- 认证信息**不会存储**在 Skill 或系统日志中
- 认证过程**通过加密通道**传输（HTTPS/LDAP over SSL）
- Agent 不会向第三方披露用户的学术身份信息

**安全建议**：
- **优先使用机构订阅**而非个人账号（避免个人凭据泄露）
- **避免在对话中提供密码**，建议使用临时访问令牌
- 如果有 VPN/校园网访问，优先使用而非直接提供账号凭据
- Agent 完成检索后，建议**及时撤销临时授权**

## 7 非功能需求

### 7\.1 性能需求

- 需求解析响应速度：1秒内完成意图识别与参数提取

- 检索输出时效：单次检索整体响应不超过10秒

- 单次输出文献数量：默认输出5\-8篇优质论文，可支持用户自定义数量

### 7\.2 准确性需求

- 意图识别准确率≥95%，杜绝非检索场景误触发

- 文献匹配精准，无大量无关、跨领域无效文献

- 论文信息提取准确，引用格式规范无明显错误

### 7\.3 可用性与可迭代性

- Skill 模块化独立存在，可单独开启/关闭，不影响 Agent 其他能力

- 支持后续迭代新增检索数据源、新增筛选规则、优化输出格式

### 7\.4 约束与安全需求

- 仅做**公开学术论文检索与信息整理**，不提供论文下载、破解、侵权资源

- 不处理涉密、敏感、违规领域学术内容，识别敏感需求自动拒绝并提示

- 禁止生成虚假论文信息，无匹配文献时明确告知用户，不编造内容

## 8 技能运行流程设计

完整闭环流程，适配 Hermes Agent 调度逻辑：

1. **意图识别**：Agent 接收用户输入，判断是否命中论文检索场景

2. **需求解析**：提取检索主题、语种、时间、筛选条件等核心参数

3. **智能检索**：基于参数执行学术文献检索，抓取合规公开文献信息

4. **筛选排序**：过滤无效、重复文献，按学术价值优先级排序

5. **信息规整**：标准化提取论文字段，生成规范参考文献格式

6. **结果输出**：按照固定结构输出检索结果\+领域总结

7. **迭代应答**：支持用户二次筛选、补充检索、细化解读需求
