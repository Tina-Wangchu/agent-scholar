## 📋 标准化的交互模板

以后当Agent检索结果为0时，应该这样回复：

```markdown
📊 检索结果：0篇论文

🔍 检索参数：
- 主题：人工智能
- 时间范围：近1年 (2025-07-01 至 2026-07-01)
- 关键词：AI, machine learning
- 数据源：Semantic Scholar, arXiv, CrossRef

💡 可能的原因：
- 时间范围过窄（该时间范围内相关论文较少）
- 关键词过于具体（匹配不到相关文献）
- 该领域近期发表论文较少

🎯 建议的调整方案：
1. 扩大时间范围：3年 / 5年 / 10年 / 不限年份
2. 调整关键词：更通用的术语
3. 更换检索策略

❓ 请选择：
- "扩大到3年" / "扩大到5年" / "不改，我要最新的" / "调整关键词为xxx"
```

## 🎯 为什么这个原则很重要？

1. **用户意图明确**：您指定"近一年"是有原因的（如只需要最新研究）
2. **透明度原则**：您有权知道何时您的约束条件无法满足
3. **控制权原则**：您可能更偏好"无结果"而非"较旧的论文"
4. **信任基础**：擅自修改参数会破坏您对Agent的信任

## 💡 给用户的建议

### 如何避免此类问题

1. **明确您的需求**：清楚地说明时间范围的重要性
   - ✅ "我要近一年的论文，不要更旧的"
   - ❌ "找一些AI论文"（太模糊）

2. **理解Agent的限制**：Agent应该询问，而非擅自决定

3. **提供反馈**：如果Agent擅自修改参数，立即指出问题
   - ✅ "我要求的是近一年，为什么给了我5年的论文？"

### 如何验证Agent是否遵守原则

测试对话：
```
You: "检索近1天的transformer论文"
Agent应该："⚚️ 近1天内未找到相关论文。是否扩大时间范围？"
Agent不应该：[自动改为1年并返回结果]
```

## 🎓 学到的经验

您的这个反馈非常重要，它揭示了一个更深层的Agent交互设计问题：

**Agent vs 用户的关系**
- ❌ **错误模式**：Agent = 替用户做决定的"智能助手"
- ✅ **正确模式**：Agent = 透明沟通的"协作伙伴"

**核心区别**：
- 错误：Agent认为"用户会满意结果"所以擅自修改
- 正确：Agent理解"用户约束是明确指令"，必须尊重并沟通

这类似于医生与病人的关系：
- ❌ 医生："这个药没用，我直接给你换个药"
- ✅ 医生："这个药效果不理想。我建议换另一种，您觉得如何？"
