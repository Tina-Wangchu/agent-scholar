# Hermes Agent 内置 Skills 测试攻略

## 概述

Hermes Agent 框架提供了多个内置skills，可以通过 `/skill-name` 命令直接调用。本文档提供详细的测试攻略，帮助你快速上手和验证各个skills的功能。

---

## 内置 Skills 列表

| deep-research | `/deep-research` | 深度网络搜索，多源验证，生成引用报告 |

### 阶段2：代码质量测试

#### 3️⃣ 测试 `code-review` skill（代码审查）

**目的**：验证能否发现代码中的bug和优化点

**准备工作**：
1. 创建一个包含明显bug的测试文件
2. 或修改现有文件引入问题

**测试文件示例**：
```python
# test_bugs.py
def divide(a, b):
    return a / b  # 没有除零检查

result = divide(10, 0)
print(result)
```

**测试步骤**：
```
/code-review
```

**可选参数**：
- `--low` - 较少的检查，高置信度
- `--medium` - 中等检查
- `--high` - 广泛检查，包含不确定的发现
- `--max` - 最全面检查
- `--comment` - 将发现作为PR评论发布
- `--fix` - 自动应用修复

**预期结果**：
- 发现除零错误
- 建议添加错误处理
- 可能建议添加类型提示

**验证点**：
- ✅ 成功识别bug
- ✅ 提供修复建议
- ✅ 使用 `--fix` 时自动修复

---

#### 4️⃣ 测试 `simplify` skill（代码简化）

**目的**：验证能否优化代码的简洁性和效率

**准备工作**：
```python
# test_complex.py
def calculate_sum(numbers):
    result = 0
    for i in range(len(numbers)):
        num = numbers[i]
        if num > 0:
            result = result + num
    return result
```

**测试步骤**：
```
/simplify
```

**预期结果**：
- 简化为使用 `sum()` 和列表推导式
- 移除不必要的变量

**验证点**：
- ✅ 代码更简洁
- ✅ 功能保持不变
- ✅ 更符合Python惯用法

## 常见问题排查

### Q1: Skill调用失败

**可能原因**：
- Skill名称错误
- 缺少必要参数
- 权限不足

**解决方案**：
- 检查skill拼写
- 查看skill文档
- 检查权限设置

---

### Q2: `/run` 无法启动项目

**可能原因**：
- 项目类型无法识别
- 缺少启动脚本
- 依赖未安装

**解决方案**：
- 确保有package.json/requirements.txt等配置文件
- 检查是否有启动脚本
- 先安装依赖

---

### Q3: `/code-review` 没有发现问题

**可能原因**：
- 代码质量已经很好
- 检查级别不够高
- 没有待审查的变更

**解决方案**：
- 使用 `--high` 或 `--max` 提高检查级别
- 确保有staged或unstaged的变更
- 尝试引入明显bug进行测试
